1. Co to vlastně řešíme?

Vítejte u dnešní online lekce! Minule jste si založili GitHub a nahráli tam svůj web. Dnes se podíváme pod pokličku toho, jak to vlastně funguje a jaké nástroje k tomu používáme.

Verzování vs. Bugtracking

V profesionálním vývoji softwaru se neustále setkáváte se dvěma pojmy, které se rády pletou:

  • Verzovací systém (např. Git): Nástroj, který sleduje historii změn vašich souborů. Umožňuje "cestování v čase" a týmovou spolupráci bez přepisování práce kolegů.
  • Bugtracking systém (např. Jira, Trello, GitHub Issues): Zápisník chyb a úkolů. Zde si tým eviduje věci jako: "Tlačítko na hlavní straně nefunguje" nebo "Přidat sekci kontakty".
💡 Častý omyl: Git samotný NENÍ bugtrackingový systém. GitHub je platforma, která používá Git pro verzování, ale navíc k němu přidává vlastní bugtracking (záložka Issues) a další nástroje.

2. Proč potřebujeme Git?

Představte si, že píšete web, ročníkovou práci nebo jakýkoliv větší projekt. Pravděpodobně znáte tento systém zálohování souborů:

  • index.html
  • index_oprava.html
  • index_FINAL.html
  • index_FINAL_opravdu_hotovo_3.html

Tohle je samozřejmě v praxi neudržitelné. Co když s vámi na webu pracuje spolužák? Jak spojíte jeho kód s vaším, aniž byste museli kopírovat řádky textu přes Messenger?

Git tohle všechno řeší. Soubor máte na disku vždy jen jeden, ale Git si v pozadí pamatuje úplně každou uloženou změnu (verzi), ví kdo ji udělal, kdy ji udělal a proč ji udělal.

3. Rozdíl mezi Gitem a GitHubem

Často se tyto dva pojmy zaměňují. Představte si to jako rozdíl mezi motorem v autě a parkovacím domem.

Git = Motor (Nástroj)

Je to program, který běží přímo u vás v počítači. Funguje zcela offline. Nemá žádné grafické prostředí na webu, ovládá se většinou přes příkazovou řádku (terminál) nebo přes rozšíření v editoru (např. VS Code). Sám o sobě hlídá změny u vás na disku.

GitHub = Parkovací dům (Platforma)

Je to webová služba (server vlastněný Microsoftem). Můžete tam svůj lokální Git repozitář nahrát (zaparkovat). Díky tomu k němu mají přístup ostatní, je zálohovaný v cloudu a jak jste si zkusili minule, přes GitHub Pages z něj jde rovnou udělat živý web.

4. Jak Git přemýšlí: 3 Stavy

Abychom mohli Git používat, musíme pochopit, jak pracuje se soubory. Vaše soubory putují přes tři pomyslné stanice:

  1. Working Directory (Pracovní adresář): Zde tvoříte a upravujete. Máte otevřený kód, píšete, ukládáte. Git si říká: "Aha, vidím změnu, ale zatím ji neřeším."
  2. Staging Area (Přípravna): Když si myslíte, že je část práce hotová, "přidáte" ji na pódium. Říkáte tím: "Gite, tuhle úpravu si připrav, v další verzi ji chci zapsat."
  3. Repository (Historie): Finální potvrzení (tzv. Commit). Soubory ze Staging Area se trvale zapíší do historie Gitu s vaším popiskem (např. "Změna barvy tlačítka"). Odsud už se dá kdykoliv vracet v čase.

5. Nejdůležitější Git příkazy

Tyto příkazy se zadávají do terminálu a slouží pro práci na vašem počítači (offline):

git init
# Zapne Git ve vaší složce. Vytvoří skrytou složku .git a z obyčejné složky je rázem repozitář.

git status
# Váš nejlepší kamarád. Vypíše, jaké soubory jsou změněné a v jakém jsou stavu.

git add .
# Přesune VŠECHNY (tečka znamená vše) upravené soubory do Staging Area (přípravny).

git commit -m "Oprava barvy nadpisu"
# Uloží připravené soubory do trvalé historie a přidá k nim vysvětlující vzkaz.

6. Komunikace se světem (GitHub)

Zatím jsme ukládali historii jen u vás na disku. Pro sdílení světu používáme tyto příkazy:

git push
# Vezme vaši lokální (počítačovou) historii a "zatlačí" (odešle) ji na vzdálený server (GitHub).

git pull
# Pokud kolega udělal změny na GitHubu, vy si je musíte "přitáhnout" k sobě do PC.

git clone https://github.com/vase-jmeno/repozitar.git
# Stáhne celý existující projekt z GitHubu přímo k vám na disk.

7. Samostatné cvičení

Vzhledem k mé nepřítomnosti máte na dnešní hodinu samostatný praktický úkol. Budete navazovat na minulou hodinu.

Váš dnešní úkol:

  1. Otevřete si na počítači terminál (nebo terminál přímo ve VS Code).
  2. Stáhněte si svůj minulý web z GitHubu do PC příkazem:
    git clone https://repozitar.cz/
  3. Otevřete staženou složku ve VS Code a proveďte viditelnou změnu ve vašem index.html nebo CSS.
  4. Otevřete terminál ve VS Code (musíte být uvnitř složky!) a připravte soubory: git add .
  5. Uložte změnu do historie: git commit -m "Aktualizace webu z lokálu"
  6. Odešlete to na GitHub: git push
  7. Zkontrolujte svůj veřejný GitHub Pages web. Za pár minut by se měla úprava projevit!
