Co je Git?
Systém pro správu verzí (Version Control System)

📸 Snímky kódu

Git si průběžně ukládá fotografie tvého projektu. Kdykoli se vrátíš do libovolného bodu v historii.

🕒 Časová osa

Každá změna má autora, datum a popis. Vždy víš, kdo, kdy a proč něco změnil.

🌿 Větve

Pracuj na nových funkcích odděleně, aniž by rozbíjely hlavní kód. Větve jsou v Gitu levné a rychlé.

🤝 Spolupráce

Více vývojářů může pracovat na jednom projektu současně bez konfliktů.

Git běží lokálně na tvém počítači — nepotřebuje internet.
Co je GitHub?
Cloudové místo pro tvé Git repozitáře
# Git (lokálně)
$ git init
$ git commit

# Na tvém PC 💻

Git je nástroj nainstalovaný na počítači. Pracuje offline, ukládá historii lokálně.

# GitHub (cloud)
$ git push origin main

# Na internetu 🌐

GitHub je platforma kde sdílíš kód, spolupracuješ s ostatními a hostíš projekty.

Záloha kódu Spolupráce Open Source Portfolio
git init
Začínáme — vytvoříme Git repozitář
# 1. Vytvoř složku projektu a vstup do ní
$ mkdir muj-projekt
$ cd muj-projekt

# 2. Inicializuj Git repozitář
$ git init

# Výstup:
Initialized empty Git repository in .git/
1
Vytvoří se skrytá složka .git/ — tady Git ukládá celou historii projektu. Nikdy ji ručně nemaž!
2
Složka se stane lokálním repozitářem — Git od teď sleduje všechny změny souborů.
git add & git commit
Jak ukládáme změny do historie
# Přidej konkrétní soubor do "staging area"
$ git add index.html

# Nebo přidej VŠECHNY změněné soubory najednou
$ git add .

# Potvrď změny s popisem (commit)
$ git commit -m "Přidána navigace na homepage"

📦 git add

Přesuneš soubory do staging area — jako balíček připravený k odeslání. Git je ještě NEULOŽIL do historie.

💾 git commit

Teprve commit uloží snapshot do trvalé historie. Vždy piš výstižnou zprávu — tvé budoucí já ti poděkuje!

git status & git log
Co se děje v mém repozitáři?
$ git status

On branch main
Changes not staged:
modified: index.html

Untracked files:
style.css

nothing added to commit

Zobrazí, které soubory jsou změněné, ve staging nebo nové.

$ git log --oneline

a3f2c1b Oprava chyby v login formuláři
9d8e7f6 Přidán dark mode
2b1a0c9 Inicializace projektu

Vypíše historii commitů. Každý má unikátní hash ID a zprávu.

push · pull · clone
Komunikace s GitHubem
⬇
git clone  — stáhni celý repozitář z GitHubu poprvé
$ git clone https://github.com/uzivatel/repozitar.git
⬆
git push  — nahraj lokální commity na GitHub
$ git push origin main
🔄
git pull  — stáhni nejnovější změny od ostatních
$ git pull origin main
Zlaté pravidlo: vždy začínaj den s git pull — budeš mít nejnovější verzi kódu!
🌿 Větve (Branches)
Pracuj na funkci bez narušení hlavního kódu
# Zobraz větve
$ git branch
* main

# Vytvoř novou větev a přepni se do ní
$ git checkout -b feature/nova-funkce

# (modernější způsob)
$ git switch -c feature/nova-funkce

# Přepnutí zpět na main
$ git switch main

🛡️ main

Hlavní větev — vždy by měla obsahovat funkční kód. Nikdy tam commituj přímo!

🔨 feature/...

Nová funkce = nová větev. Konvence pojmenování: feature/popis, fix/chyba

📅 Každodenní workflow
Jak vypadá typický pracovní den s Gitem
1
git pull  — Stáhni nejnovější změny od kolegů
2
git switch -c feature/nova-vec  — Vytvoř vlastní větev
3
✍️ Pracuj na kódu... píši, ladím, testuji
4
git add .  +  git commit -m "..."  — Ulož průběžné změny
5
git push origin feature/nova-vec  — Nahraj na GitHub
6
🔀 Otevři Pull Request na GitHubu a požádej o code review
🔀 Merge & Konflikty
Spojování větví dohromady
# Přepni se na main a slij do ní svou větev
$ git switch main
$ git merge feature/nova-funkce

# Pokud nastane konflikt — Git ti řekne:
CONFLICT (content): Merge conflict in index.html

# Otevři soubor, vyřeš konflikt, pak:
$ git add index.html
$ git commit -m "Vyřešen konflikt merge"
Konflikt nastane, když dva lidé změní stejný řádek. Git to označí v souboru — ty se rozhodneš, která verze zůstane. Je to normální a zvládneš to! 💪

🌍 Kontribuce & Issues
Jak přispívat do open source projektů
1
Fork — Udělej si vlastní kopii repozitáře cizího projektu na svůj GitHub účet
2
git clone — Stáhni svůj fork lokálně a pracuj na změně
3
Oprav chybu / přidej funkci → commit + push do svého forku
4
Otevři Pull Request do originálního projektu — autoři rozhodnou, zda kód přijmou
Issues = tikety s chybami nebo nápady. Najdeš je na každém repozitáři. Hledej štítek good first issue — ty jsou pro nováčky! 🎯
🌐 GitHub Pages
Zdarma hostuj webové stránky přímo z repozitáře
1
Vytvoř repozitář s názvem uzivatelske-jmeno.github.io
2
Nahraj soubory HTML/CSS/JS a udělej git push
3
V Settings → Pages zvol zdroj (větev main, složka /root)
4
Za chvíli běží web na https://uzivatelske-jmeno.github.io 🎉
Skvělé pro portfolio, projektové dokumentace nebo jednoduché weby. Zdarma, bez serveru, bez složité konfigurace.
